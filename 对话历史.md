## 时间线与关键决策

### 阶段1：计划与实现（2026-03-26）

1. **任务**：将2D风场超分（U10/V10，2通道）扩展为3D风场超分（6层×u/v/w=18通道）
2. **参考**：FuXi-CFD论文（arXiv:2505.12732），ViT做3D风场降尺度
3. **关键设计决策**：
   - 6个eta层：ml0(~10m), ml1(~30m), ml2(~50m), ml3(~70m), ml5(~120m), ml10(~300m)
   - 10个条件变量：t2, z, lu, tsk, swdown, glw, hfx, lh, psfc, pblh
   - UNet输入28ch（18 LR + 10条件），输出18ch
   - 降采样因子=4（180→45→180）
   - 新建Dataset3dWind子类，仅覆盖`_get_var_name`返回完整变量名（不截断）
   - 新建ExperimentSchrodingerBridge3dWindConfig实验配置
   - 训练脚本添加`--experiment_name`参数

### 阶段2：数据准备

4. **Arakawa C网格去交错**：WRF的U/V/W在交错网格上，需取相邻两点平均对齐到质量点网格
5. **数据量**：520个npz文件（约21.7天），每个46个数组（36风场+10条件）
6. **biases/scales**：运行`prepare_wind_data_3d.py`后计算，已填入`config_wind_3d.yml`

### 阶段3：训练调优

7. **路径修改**：服务器代码位置从 `.../tzq/schrodinger-bridge-sr-micrometeorology` 改为 `.../tzq/git/schrodinger-bridge-sr-micrometeorology`，需修改`prepare_wind_data_3d.py`和`prepare_wind_data.py`的`OUTPUT_DIR`
8. **hr_data_shape变更记录**：
   - 最初计划：128×128（和2D版一样）
   - 改为192×192（>180不丢信息，且能被32整除）
   - 后来为加速训练又改回128×128
9. **模型容量变更**：
   - 最初计划：inner_channel=64, res_blocks=2（比2D版大）
   - 后来为加速CPU训练改回：inner_channel=32, res_blocks=1（和2D版一样）
   - 计算量约为原计划的1/8
10. **use_amp**：从true改为false（CPU不支持AMP混合精度）
11. **SLURM脚本**：`task-wind-3d-train.sh`，注意Windows换行符问题需`sed -i 's/\r$//'`

### 阶段4：训练完成与问题排查

12. **训练结果**：400 epoch，最佳loss=0.0410（epoch 388），总耗时约22.75小时
13. **推理报错**：`KeyError: 'hr_tm'` — 训练脚本推理部分写死了温度变量名`"hr_tm"`
    - **修复**：改为遍历`config.data.target_variable_names`对每个通道分别反标准化
14. **可视化问题**：
    - colorbar和图重叠 → 改用`fig.subplots_adjust(right=0.88)` + `fig.add_axes()`精确定位
    - W分量全白 → 色标从极值改为95百分位截断
    - 垂直剖面图中LR比预测更接近HR → 因为剖面取平均抹掉了空间细节，LR均值本就保留好，该指标不适合评价超分
15. **组会报告**：写在`组会报告-3D风场超分.md`

## 常用命令

```bash
cd /public3/home/scg4074/tzq/git/schrodinger-bridge-sr-micrometeorology
python prepare_wind_data_3d.py
```

### 创建数据软链接（服务器）
```bash

ln -s /public3/home/scg4074/tzq/git/schrodinger-bridge-sr-micrometeorology/prepare_npz_wind_3d data/DL_data/wrf_3d_v1
```

### 训练（服务器）
```bash
sbatch task-wind-3d-train.sh
# 或手动：
python scripts/train_schrodinger_bridge_model.py \
  --config_path configs/config_wind_3d.yml \
  --experiment_name ExperimentSchrodingerBridge3dWind \
  --device cpu
```

### 可视化（服务器）
```bash

python visualize_wind_3d.py \
  --npz_dir data/DL_data/wrf_3d_v1 \
  --output_dir visualizations_wind_3d \
  --config_path configs/config_wind_3d.yml \
  --model_weight_path data/DL_result/ExperimentSchrodingerBridge3dWind/config_wind_3d/checkpoint.pth \
  --num_samples 3

# 仅验证数据（不需要模型）
python visualize_wind_3d.py \
  --npz_dir data/DL_data/wrf_3d_v1 \
  --data_only --num_samples 1
```
