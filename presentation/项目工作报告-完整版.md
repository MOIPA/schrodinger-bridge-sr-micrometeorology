# 项目工作报告：基于薛定谔桥的微气象超分辨率重建

---

## 一、项目概述

### 1.1 研究目标

利用深度学习生成式模型（薛定谔桥 Schrödinger Bridge），从低分辨率气象场快速重建高分辨率气象场。研究区域为香港及珠三角地区（112.98°-114.65°E, 21.84°-23.39°N），数据来源为WRF中尺度气象模式d04域输出（水平分辨率~1km，180×180网格）。

### 1.2 技术路线

采用**薛定谔桥**（Schrödinger Bridge）作为生成框架，以UNet为骨干网络。薛定谔桥是一种基于最优传输理论的生成式建模方法，建模从低分辨率分布到高分辨率分布的概率传输路径。

与传统扩散模型（从随机噪声出发）相比，薛定谔桥的优势在于：
- 起点是低分辨率场（包含有用的先验信息），而非随机噪声
- 传输路径更短，采样效率更高
- 天然适合配对的超分辨率任务

### 1.3 参考论文

**FuXi-CFD**（arXiv:2505.12732）：*An AI-Based Downscaling Framework for Generating Detailed Three-Dimensional Wind Fields*

| 项目 | FuXi-CFD | 我们的方案 |
|------|----------|-----------|
| 生成框架 | 判别式模型（Vision Transformer） | 生成式模型（薛定谔桥） |
| 训练数据 | 12000+ CFD(RANS)模拟 | WRF模式输出 |
| 输出 | u, v, w, k × 27层 = 108通道 | u, v, w × 6层 = 18通道 |
| 精度 | 水平风<7%误差，垂直风~20% | 待完善定量评估 |

**我们的差异化**：使用生成式框架（非判别式）+ 引入丰富的地表物理条件变量。

## 二、完整工作历程

项目经历了三个阶段，逐步从单变量扩展到多变量多层：

```
阶段一（2D温度超分）→ 阶段二（2D风场超分）→ 阶段三（3D风场超分）
    1通道输出            2通道输出              18通道输出
```

### 阶段一：2D温度超分辨率（基础验证）

#### 目标
验证薛定谔桥框架在气象超分任务上的可行性。

#### 配置
| 项目 | 值 |
|------|-----|
| 目标变量 | 2米气温（T2），1通道 |
| 条件变量 | 20个（10个地表变量 + 9个模式层变量 + 1个LR温度） |
| UNet输入 | 21通道 |
| UNet输出 | 1通道 |
| 训练轮次 | 1001 epochs |
| 批大小 | 4 |
| 学习率 | 0.0001 |

#### 消融实验

为理解不同类型条件变量的贡献，设计了4组消融实验：

| 实验 | 移除的变量 | 输入通道 | 研究目的 |
|------|----------|---------|---------|
| **Baseline（全变量）** | 无 | 21 | 完整模型基准 |
| **去辐射** | swdown, glw | 19 | 辐射变量对温度超分的贡献 |
| **去地形** | z(HGT), lu(LU_INDEX) | 19 | 地形和土地利用的重要性 |
| **去地表物理** | tsk, hfx, lh, psfc, pblh | 15 | 地表通量和边界层信息的作用 |

#### 阶段一成果
- ✅ 验证了薛定谔桥框架在气象超分任务上的有效性
- ✅ 通过消融实验量化了不同条件变量的贡献
- ✅ 建立了完整的数据处理→训练→推理→可视化流程

#### 目标
从单变量（温度）扩展到多变量（风场U/V分量），验证框架处理向量场的能力。

#### 阶段二成果
- ✅ 成功扩展到2通道输出，验证了框架处理向量场的能力
- ✅ 通用化的数据加载逻辑为后续3D扩展奠定基础

### 阶段三：3D风场超分辨率（当前最新工作）

#### 目标
参考FuXi-CFD论文思路，从2D风场扩展到3D多层风场重建，实现对6个垂直层的u/v/w三风分量的同步超分辨率。

#### 配置
| 项目 | 值 |
|------|-----|
| 目标变量 | 6层 × U/V/W = 18通道 |
| 条件变量 | 10个地表物理变量 |
| UNet输入 | 28通道（18 LR + 10条件） |
| UNet输出 | 18通道 |
| 训练轮次 | 400 epochs |
| 批大小 | 2 |
| 学习率 | 0.0005 |
| 空间尺寸 | 128×128 |
| inner_channel | 32 |
| res_blocks | 1 |
| 训练设备 | CPU（超算128核），约22.75小时 |

#### 目标变量：三维风场

**三个风分量**：

| 分量 | 物理含义 | 典型量级 |
|------|---------|---------|
| U（东西风） | 沿纬度方向，正值为西风 | 数~十几 m/s |
| V（南北风） | 沿经度方向，正值为南风 | 数~十几 m/s |
| W（垂直风） | 垂直方向，正值为上升 | 0.01~0.3 m/s |

**6个垂直层级**（WRF eta地形跟随坐标）：

| 层级 | 近似高度 | 物理意义 |
|------|---------|---------|
| ml0 | ~10m | 最近地面层，标准气象观测高度 |
| ml1 | ~30m | 城市建筑群平均高度 |
| ml2 | ~50m | 城市冠层顶部 |
| ml3 | ~70m | 小型风机轮毂高度 |
| ml5 | ~120m | 大型风机轮毂高度 |
| ml10 | ~300m | 边界层中上部 |

#### 条件变量：10个地表物理变量

| 变量 | 物理含义 | 对风场的影响 |
|------|---------|-------------|
| HGT（地形高度） | 地面海拔高度 | **最核心**。地形机械强迫：迎风坡上升、背风坡下沉、峡谷加速、山脊绕流 |
| LU_INDEX（土地利用） | 城市/森林/水体等分类 | 决定地表粗糙度，城市摩擦大减速、水面摩擦小加速 |
| T2（2米气温） | 近地面气温 | 温度空间差异驱动热力环流（热岛环流、海陆风） |
| TSK（地表温度） | 地表皮肤温度 | 直接热源，驱动感热通量，空间分布比T2更不均匀 |
| SWDOWN（短波辐射） | 太阳辐射 | 白天加热→对流边界层→强湍流；夜间=0→稳定边界层 |
| GLW（长波辐射） | 大气下行红外辐射 | 夜间辐射冷却→逆温层→抑制混合 |
| HFX（感热通量） | 地表→大气显热传递 | 驱动热对流，城市HFX大→热岛环流动力源 |
| LH（潜热通量） | 蒸发消耗的潜热 | 蒸发冷却效应，影响温度梯度和局地环流 |
| PSFC（地面气压） | 地面气压 | 气压梯度力是驱动风的根本动力 |
| PBLH（边界层高度） | 湍流混合的"天花板" | 决定我们的6层是在同一混合环境还是跨越混合层顶 |

#### 数据处理关键技术

**1. Arakawa C网格去交错**

WRF将U/V/W存储在网格不同位置（边界上），需取相邻两点平均对齐到网格中心：
```
U_center = (U[:,:,:-1] + U[:,:,1:]) / 2    # x方向: 181→180
V_center = (V[:,:-1,:] + V[:,1:,:]) / 2      # y方向: 181→180
W_center[k] = (W[k] + W[k+1]) / 2            # z方向: nz+1→nz
```

**2. 低分辨率模拟**

因不具备真实低分辨率数据（d01-d03已删除），采用降采样模拟：
```
HR(180×180) → avg_pool(4×4) → 中间态(45×45) → bicubic插值 → LR(180×180)
```

**3. 数据集结构**

每个时间步一个npz文件，共520个样本（约21.7天），每个文件包含46个数组：
- 18个HR风场 + 18个LR风场 + 10个条件变量

#### 训练结果

| 指标 | 值 |
|------|-----|
| 最佳epoch | 388 |
| 最佳训练loss | 0.0410 |
| loss曲线 | 稳定收敛，最终在0.04附近波动 |

#### 可视化结果

生成三类对比图：
1. **水平风场(U,V)对比**：6行(层级)×3列(LR 45×45 / HR真值 / 模型预测)，底色=风速，箭头=风向
2. **垂直速度(W)对比**：同布局，RdBu_r色标（红=上升、蓝=下沉），95百分位截断增强细节
3. **垂直剖面**：U和W随高度变化的曲线对比

**初步观察**：
- 水平风场(U,V)：模型能恢复部分高分辨率细节纹理，大尺度模式保持良好
- 垂直速度(W)：量级极小（比U/V小1-2个数量级），大部分区域接近0，模型学习存在困难
- 预测结果存在一定"颗粒感"——这是生成式模型采样过程中随机性引入的高频噪点

## 四、模型架构总结

所有阶段共享相同的底层架构：

```
                    ┌─────────────────┐
                    │  NPZ数据(180²)  │
                    └────────┬────────┘
                             │ resize to 128×128
                    ┌────────┴────────┐
                    │                 │
              LR目标场(N_t ch)  条件变量(N_c ch)
                    │                 │
                    │   ┌─────────────┘
                    │   │
              ┌─────▼───▼─────┐
              │cat→(N_t+N_c)ch│
              │               │
              │  UNet + SI    │  ← 薛定谔桥（随机插值）
              │               │
              └───────┬───────┘
                      │
                HR预测 (N_t ch, 128×128)
```

**UNet配置**：
- channel_mults: [1, 2, 4, 8, 8]（5层下采样）
- inner_channel: 32
- attention: 在16×16分辨率处
- 随机插值(SI): 10时间步, quadratic公式

## 五、当前存在的问题

### 5.1 数据层面

1. **训练数据为自降采样**：LR由HR降采样模拟而非真实低分辨率模式输出（d01-d03数据因存储空间不足已删除），可能高估实际应用效果
2. **空间尺寸有损**：模型在128×128上训练，原始数据为180×180，resize过程丢失空间信息。应改为192×192（>180且能被32整除）
3. **数据量有限**：仅520个样本（约21.7天），对18通道输出任务偏少

### 5.2 模型层面

4. **模型容量受限**：为在CPU上可行训练，使用了较小配置（inner_channel=32, res_blocks=1），可能不足以学好18通道输出
5. **W分量学习困难**：W的量级比U/V小1-2个数量级（标准差0.15 vs 4.4），在统一的L2 loss中被U/V主导，W的实际学习效果可能很差
6. **颗粒感/噪点**：生成式模型采样过程引入的随机性导致预测结果有高频噪点，尤其在模型容量小、采样步数少时更明显

### 5.3 评估层面

7. **缺少定量评估**：目前仅有训练loss和定性可视化，缺乏逐变量的RMSE、MAE、SSIM等定量指标
8. **缺少基线对比**：没有与双三次插值、纯回归UNet等基线方法做对比

## 六、未来改进方向

### 6.1 近期可执行

| 改进项 | 具体操作 | 预期效果 |
|--------|---------|---------|
| **增大模型容量** | inner_channel: 32→64, res_blocks: 1→2 | 提升18通道的学习能力，减少颗粒感 |
| **空间尺寸优化** | hr_data_shape: 128→192 | 不丢失信息（192>180），更准确的超分重建 |
| **启用混合精度** | use_amp: true | 加速训练，降低显存占用 |
| **增加采样步数** | n_timestep: 10→20或更多 | 采样更充分，减少噪点/颗粒感 |
| **定量评估体系** | 在test集上逐变量计算RMSE/MAE/SSIM | 客观评价每个通道的超分质量 |

### 6.2 中期改进

| 改进项 | 具体操作 | 预期效果 |
|--------|---------|---------|
| **W分量加权** | 在loss中对W通道增加权重（如×10），或对W独立标准化 | 改善W的学习效果 |
| **基线对比实验** | 实现bicubic插值baseline + 纯回归UNet baseline | 客观展示薛定谔桥的优劣势 |
| **增加消融实验** | 参照阶段一的消融设计，对3D风场做条件变量消融 | 量化各条件变量对3D风场的贡献 |
| **增加训练数据** | 使用更多时间段的WRF输出 | 提升模型泛化能力 |

### 6.3 长期方向

| 改进项 | 说明 |
|--------|------|
| **增加垂直层数** | 从6层扩展到更多层（参考FuXi-CFD的27层），更密集覆盖边界层 |
| **增加输出变量** | 加入温度T、湍动能TKE等变量，实现更完整的3D大气场重建 |
| **真实低分辨率数据** | 使用WRF的d01/d02/d03粗分辨率域作为输入（需要重新运行WRF模拟保存输出），更接近实际应用 |
| **物理约束损失** | 加入质量守恒（散度约束）、动量守恒等物理约束作为正则化项 |
| **频域损失** | 参考FuXi-CFD的频域损失设计，提升高频细节重建的准确性 |
| **多区域泛化** | 在不同城市/地形条件下训练和测试，验证模型的迁移能力 |

## 九、总结

本项目从2D温度超分出发，经过三个阶段迭代，已成功实现基于薛定谔桥的**3D多层风场超分辨率重建**（6层×3分量=18通道输出）。当前模型在CPU上完成了400 epoch训练，loss收敛至0.04，水平风场可视化效果合理。

**核心价值**：
- 验证了薛定谔桥生成式框架在多通道气象超分任务上的可行性
- 建立了完整的从数据准备到训练推理的技术流程
- 引入了丰富的物理条件变量，为物理可解释的超分辨率提供了基础

**下一步重点**：增大模型容量、优化空间尺寸、建立定量评估体系、添加基线对比实验，向论文级别的结果推进。
